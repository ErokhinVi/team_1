# Контракт блока backend

Сюда вписывай ручки, которые твой блок отдаёт наружу. Соседи по команде
видят только этот файл — не код. Если ручка изменилась или появилась новая —
обнови этот файл, иначе сосед о ней не узнает.

## Что я отдаю наружу

### GET /health
Проверка живости. Возвращает `{status, team, block, commit, clients_loaded,
transactions_loaded}`.

### GET /coin-flip
Подбрасывание монетки: честный случайный выбор между орлом и решкой
(на базе `secrets.randbits` — криптостойкий генератор). Параметров нет.
Возвращает `{result, result_en, ts}`, где `result` — `"орёл"` или
`"решка"`, `result_en` — `"heads"` или `"tails"`, `ts` — ISO-таймстемп
броска.

### GET /casino/roulette
Спин европейской рулетки. Параметров нет. Возвращает
`{number, color, parity, ts}`. `number` — целое 0..36; `color` —
`"красный" | "чёрный" | "зелёный"` (зелёный только для нуля); `parity` —
`"чёт" | "нечёт" | "ноль"`.

### GET /casino/dice
Бросок кубиков. Параметр `count` (опционально, по умолчанию 2,
максимум 10) — сколько кубиков бросить. Возвращает
`{count, rolls, sum, ts}`, где `rolls` — массив значений 1..6.

### GET /casino/slot-machine
Однорукий бандит. Параметров нет. Возвращает `{reels, jackpot, outcome, ts}`,
где `reels` — массив из трёх символов из набора
`["вишня", "лимон", "апельсин", "колокольчик", "звезда", "семёрка"]`,
`jackpot` — bool (все три совпали), `outcome` —
`"джекпот" | "две совпали" | "мимо"`.

### GET /casino/blackjack/draw
Тянем карты из свежей перетасованной колоды (52 карты). Параметр `count`
(опционально, по умолчанию 2, максимум 10). Возвращает
`{hand, total, blackjack, bust, ts}`. `hand` — массив `{rank, suit}`
(`rank` ∈ `["A","2"…"10","J","Q","K"]`, `suit` ∈ `["♠","♥","♦","♣"]`).
`total` — сумма по правилам блэкджека (тузы автоматически складываются как
1, если иначе перебор). `blackjack` — `true`, если `count == 2` и сумма
ровно 21. `bust` — `true`, если сумма больше 21.

### GET /clients
Список клиентов команды. Параметры запроса (все опциональные):
- `segment` — строка, сегмент клиента;
- `has_overdue` — bool, был ли просрочен платёж;
- `min_income` — int, минимальный доход в рублях;
- `limit` — int, ограничение по числу записей (по умолчанию 50, максимум 500).

Возвращает `{total, items: [клиенты]}`. Клиент — JSON с полями из seed:
`id, name, segment, balance_rub, income_rub, has_overdue_history` и другими.

### GET /clients/{client_id}
Полная карточка одного клиента. Возвращает объект клиента. `404`, если не найден.

### GET /transactions/{client_id}
Транзакции клиента, новые сверху. Параметры: `limit` (по умолчанию 20).
Возвращает `{total, items: [транзакции]}`. Транзакция —
`{id, client_id, type, amount_rub, ts, counterparty}`.

### POST /api/transfer
Перевод средств между клиентами команды. Принимает JSON
`{from_client_id, to, amount_rub}`. `to` — это либо id клиента, либо часть
имени получателя (поиск по подстроке). Возвращает `{status, kind
(internal|external), amount_rub, to, from_client_id, new_balance_rub, tx_id, ts}`.
